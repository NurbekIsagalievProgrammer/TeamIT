Тестовое задание: Создание простого REST API для управления задачами

Задание
Необходимо разработать простое RESTful API для управления задачами. API должно поддерживать следующие функции:
Получение всех задач
GET /api/tasks — возвращает список всех задач.
Создание новой задачи
POST /api/tasks — принимает данные задачи (название, описание, статус) и сохраняет её в базе данных.
Обновление существующей задачи
PUT /api/tasks/{id} — обновляет информацию о задаче по ID (например, название или описание).
Удаление задачи
DELETE /api/tasks/{id} — удаляет задачу по ID.
Получение задачи по ID
GET /api/tasks/{id} — возвращает информацию о задаче по её ID.
Требования:
Использование Symfony — фреймворк для разработки.
Doctrine ORM для работы с базой данных.
Использование Postman или аналогичного инструмента для тестирования API.
Валидация входных данных для создания и обновления задач (например, проверка, что название задачи не пустое).
Логирование ошибок и исключений.
Для хранения задач использовать реляционную базу данных PostgresSQL.
Ответы API должны быть в формате JSON.
Бонусные задачи (не обязательные, но приветствуются):
Реализовать пагинацию для запроса всех задач.
Добавить возможность фильтрации задач по статусу (например, только "в процессе" или только "завершённые").
Написать простые юнит-тесты для контроллеров и сервисов.
Реализовать аутентификацию с использованием JWT для безопасности API.
